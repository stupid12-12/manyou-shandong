# 漫游山东

## 项目简介

《漫游山东》是一款面向情侣自由行的动漫互动旅行攻略网页，以 2026 年 7 月 31 日至 8 月 4 日的山东之旅为主线，通过一张可交互的山东地图串联青岛、泰安和济南三座城市。

用户可以点击地图上的城市进入当地手绘地图，查看行程中的景点、美食和游玩路线。选择景点后，动漫情侣角色会沿预设路线跑向目的地，并展示门票、交通、开放时间、推荐玩法、拍照位置、省钱技巧及避坑提醒。网页重点呈现银沙滩赶海、市场购买海鲜并交给啤酒屋加工、红门徒步登泰山和济南泉水老城漫游等特色体验。

项目采用原创动漫水彩与旅行手账风格，将地图探索、角色动画和旅行攻略结合，让传统文字行程变成一段可以点击、游览和逐步打卡的互动旅程。整体设计优先适配手机端，并支持账号登录、城市切换、景点打卡、云端进度保存和低动态模式，以低预算、少绕路和当地特色为核心，为情侣提供直观、有趣且实用的山东旅行指南。

## 技术栈

本项目采用前后端分离架构。前端负责地图、角色动画和攻略交互，后端负责身份认证、攻略数据读取、打卡记录与旅行进度同步，MySQL负责持久化业务数据。

| 模块 | 技术选型 | 用途 |
| --- | --- | --- |
| 工程管理 | pnpm Workspace | 管理前端、后端和共享类型包 |
| 前端框架 | React + TypeScript | 构建页面、组件和交互逻辑 |
| 构建工具 | Vite | 提供开发服务器和生产构建 |
| 页面路由 | React Router | 管理山东总览、城市地图和景点深层链接 |
| 地图渲染 | SVG + D3.js（d3-geo）+ TopoJSON | 绘制山东及城市地图、景点和路线 |
| 动画效果 | Motion + SVG路径动画 | 实现地图转场、景点反馈和人物移动 |
| 角色动画 | CSS精灵图，后续可升级为Rive | 实现奔跑、挥手、拍照和击掌动作 |
| 页面样式 | CSS Modules + CSS Variables | 实现动漫水彩主题和响应式布局 |
| 前端状态 | React Context + useReducer | 管理当前城市、景点、动画与登录状态 |
| 后端框架 | Node.js + Express + TypeScript | 提供认证、攻略、进度和打卡API |
| ORM | Prisma | 定义数据模型、迁移和数据库访问 |
| 数据库 | MySQL 8 | 保存用户、城市、景点、攻略和进度 |
| 身份认证 | JWT + bcrypt + HttpOnly Cookie | 实现用户名密码登录和安全令牌刷新 |
| 数据校验 | Zod | 共享前后端请求、响应和配置校验规则 |
| 本地缓存 | LocalStorage | 保存界面偏好、最后访问位置和短时离线数据 |
| 自动化测试 | Vitest + Supertest + Playwright | 验证前端、API、数据库和核心流程 |
| 部署 | 静态前端 + Node服务 + MySQL | 分别部署网页、API和数据库 |

地图采用SVG而非商业地图服务，以便实现手绘动漫风格，并自由控制地图颜色、景点图标、旅行路线和人物动画。人物通过SVG路径的位置计算沿预设线路移动，避免穿过海面、建筑或山体。

用户使用用户名和密码登录。短期访问令牌保存在应用内存中，刷新令牌使用HttpOnly Cookie传输并以哈希形式保存到数据库；JWT不会长期写入LocalStorage。景点打卡和人物最终位置保存到云端，因此可以跨设备恢复。票价、开放时间、潮汐和交通等易变化信息记录来源及核验日期，不作为永久实时数据。

核心技术组合为：**React + TypeScript + Vite + Express + Prisma + MySQL + D3/SVG + Motion + JWT**。

## 核心功能

1. **山东互动地图**  
   首页展示山东地图，标记青岛、泰安和济南三座城市，并通过路线连接完整旅行顺序。点击城市即可进入对应城市地图。

2. **城市地图切换**  
   每座城市拥有独立的动漫手绘地图，展示当地景点、车站、美食区域和游览路线，支持返回山东总览及切换上一站、下一站。

3. **动漫角色移动**  
   点击景点后，情侣角色沿预设路线跑向目的地。到达后播放挥手、拍照或击掌等动作，增强旅行探索感。

4. **景点互动攻略**  
   角色到达景点后，自动展示门票、交通、游览时间、推荐玩法、拍照地点、省钱技巧、安全事项和避坑提醒。

5. **分日行程规划**  
   按 7 月 31 日至 8 月 4 日划分每日路线，用户可以切换日期，查看当天景点、游览顺序、预计时间和城市间交通安排。

6. **特色体验指南**  
   提供银沙滩赶海、市场购买海鲜、啤酒屋加工、青岛老城漫步、红门徒步登泰山和济南泉水路线等专项攻略。

7. **美食与预算信息**  
   展示各城市特色美食、推荐用餐区域、参考价格及两人消费预算，帮助用户控制旅行支出。

8. **账号与身份认证**  
   用户可以通过用户名和密码注册、登录和退出，系统使用访问令牌与刷新令牌维持安全会话。

9. **云端打卡与进度同步**  
   游览后的景点自动标记为“已打卡”，当前城市、日期和人物位置保存到MySQL，登录后可跨设备恢复。

10. **移动端适配与无障碍支持**  
    网页适配手机、平板和电脑，支持触摸操作、键盘导航、文字版行程和减少动态效果模式。

## 部署形态

- 前端构建为静态资源，部署至Vercel、Cloudflare Pages或同类平台。
- Express API部署为独立Node.js服务。
- MySQL 8部署为托管数据库或独立容器。
- 本地开发通过Docker Compose启动MySQL，并由pnpm统一运行前后端服务。
